# Implementation Alternatives Template

Produce 2-3 distinct approaches before going deeper:

```
APPROACH A: [Name]
  Summary: [1-2 sentences]
  Effort:  [S/M/L]
  Risk:    [Low/Med/High]
  Pros:    [2-3 bullets]
  Cons:    [2-3 bullets]

APPROACH B: [Name]
  Summary: ...
  Effort:  ...
  Risk:    ...
  Pros:    ...
  Cons:    ...
```

## Rules

- At least 2 approaches required
- One must be "minimal viable" (smallest diff)
- One must be "ideal architecture" (best long-term)
- Get user approval on approach before continuing to the section review
